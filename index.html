Permission Control

Gateway 權限控管 — 每個 Agent 只能做自己的事

參考 OpenClaw 架構：透過 Gateway 對每個 Agent 設定工具白名單、Exec 指令白名單、資源存取範圍

🚫 Tool ACL — 工具存取控制
每個 Agent 只能使用被核准的工具：
• 美術 Agent：只能用圖像生成工具，禁用 exec/process
• 程式 Agent：可用 exec，但只限白名單指令
• QA Agent：可執行測試，禁用 apply_patch
🔒 Exec Allowlist — 指令白名單
所有 exec 經 Gateway 代理，非白名單直接擋：
• exec-approvals.json：逐條核准可執行指令
• 每條指令記錄最後使用時間與完整命令
• Agent 無法直接存取 Shell
🛡 Per-Agent 資源隔離
• 每個 Agent 有獨立的 workspace 和記憶範圍
• 美術 Agent 不能看程式碼，程式 Agent 不能改美術素材
• 所有存取都透過 Gateway 權限檢查
👤 Per-User 個人化適配
• 同一個 Agent，對不同種子提供不同互動風格
• Gateway 依 UserID 注入不同 System Prompt
• 資深種子看詳細參數，新手看引導建議

🎯 為什麼要「實際在開發的人」？
• 他們最懂自己職種的痛點和製程細節

🔄 培育流程
• Agent主要架構師 1-on-1 帶教：先 Demo 完整流程，再讓種子實作
• 每位種子在自己的真實工作中建置 Skill

• 最終整合進 Agent，掛載至 Master Agent
關鍵：不是教他們「用 AI 工具」，是教他們把自己的製程變成 AI 可執行的 Skill，最後組裝成有權限控管的 Agent

Phased Roadmap

三階段發展藍圖 — 從單點自動化到全局調派

Phase 1：種子導入與單職種自動化
• 透過訪談選拔具備虎機/魚機開發經驗的種子人員。
• 由Agent主要架構師建置 Agent 核心與工具（MCP/API），種子負責建置專屬 Skill。
• 階段目標：各職種 Agent 能獨自完成各自的步驟流程，確立明確的階段與人工審核點。
Phase 2：Skill 自主運行與文檔連動
• 建立完善的審核與批准機制。
• 階段目標：各 Agent 在經過重要節點人工檢核後，能自行向下執行後續的 Skill。完成任務後，Agent 將自動更新專案文檔，實現流程與紀錄的無縫接軌。
Phase 3：全局統籌與跨職種調派
• 引入最高權限的 PM Agent 作為指揮中樞。
• 階段目標：由最高權限 Agent 解析大型需求，並向下調派任務給各職種 Agent，達成最終願景 — 人類只需進行戰略決策與關鍵檢核。
Key Insight

自動化記憶 + 精準 Context

不是增加工作負荷，而是在提升效率的同時訓練 Agent。
Agent 不是一次性工具，每一次的審核與修正，都在默默累積專案記憶，讓它下次做得更好。

Phase 0 — 初始化（上線前）
匯入機檯相關的設計規格、迴合邏輯、美術風格指南 → 向量化存入記憶
初始認知不完全，但足以啟動
Phase 1 — 邊做邊學（陣痛期）
日常開發中直接使用 Agent 產出，將審核的退回與修正作為最佳教材 → 邊用邊訓練
用產能的提升來覆蓋訓練成本
Phase 2 — 高效產出（成熟後）
Agent 已累積足夠歷練，新專案只需下達指令 → 自動產出 + 精準把關
實現真正的流程自動化與品質保證
